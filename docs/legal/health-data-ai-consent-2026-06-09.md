# ORSA Health Data & AI Processing Consent

Version: 2026-06-09

## 1. Purpose

This consent document explains how ORSA processes health-related information and AI-assisted workflows. It is separate from the Terms & Conditions and Privacy Policy.

## 2. Required Acknowledgements

Users acknowledge that ORSA processes health-related information, stores user health information, analyzes uploaded health documents, and uses AI systems to assist with organization and analysis.

ORSA is not a doctor, triage replacement, diagnostic system, treatment system, or emergency service.

## 3. Optional Consents

ORSA requires separate toggles for optional processing.

## 4. Memory Extraction

Users may allow ORSA to extract information from conversations. If disabled, extraction must not run.

## 5. Health Profile Generation

Users may allow GPT-OSS profile extraction. If disabled, GPT-OSS profile extraction must not run and profile updates must not be generated from conversations.

## 6. Reminder Intelligence

Users may allow ORSA to generate reminder recommendations. If disabled, reminder intelligence should not generate optional recommendations from user data.

## 7. User Controls

Users can enable consent, disable consent, revoke consent, and view current consent status from account settings.

## 8. Auditability

ORSA stores accepted version, acceptance timestamp, user ID, and document hash. Consent records are linked to acceptance records.

## 9. Versioning

This consent document is versioned independently from the Terms & Conditions and Privacy Policy.

## 10. Contact

Questions about health data and AI processing should be sent to the ORSA support contact configured for the deployed service.
